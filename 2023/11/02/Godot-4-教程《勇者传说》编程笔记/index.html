Godot 4 教程《勇者传说》编程笔记

教程视频链接：合集·《勇者传说》Godot 4教程

整理笔记的目的，主要是自己发觉跟着教程做一遍游戏 demo，并不能很好地掌握一些技术细节和设计流程，所以重新浏览一遍教程，并将比较重要的编程内容整理成笔记。

（注：部分笔记参考了瓦格良等人的博客/专栏）

00: 基础项目

0.1 修改窗口大小和拉伸模式

【操作】设置「窗口覆盖大小」为默认的视口大小（1152 * 648），「视口大小」设置为窗口覆盖大小的 1/3；将拉伸模式修改为 canvas_items

【解释】像素风格游戏，如果不修改视口大小为实际窗口覆盖大小的比例，会导致实际的游戏窗口中的一个像素，与游戏素材的中的一个像素对应（这样就会导致画面中的物体变得很小，而我们希望的是“像素风格”，素材的像素点应该明显）

【补充】canvas_items 会修改游戏的拉伸模式为“成比例放大”，如果保持默认的 disabled，拉伸后就不会放大，而且会用灰色部分填充

0.2 TileMap 的设置

【步骤】新建 tilemap 节点 - 新建 tileset - 拖入图片素材（取消自动创建图块弹窗）；设置物理层0（碰撞）

  • 快捷键：按住 shift 拖出一条直线

（Tilemap 内容详见 02 TileMap 的视频部分）

0.3 玩家场景

【步骤】sprite2d、collision、animation（关键帧包括：region_rect, hframes, frame）

【原因】只选用素材的一部分，所以需要将 region_rect 加入关键帧（因为要获取选中的区域）；水平框选动画素材（系统不知道实际的帧数），所以需要将 hframe 加入关键帧（如应用不同的素材文件，还需要将素材也加入关键帧）

【补充】素材导入的时候需要提前设置「栅格吸附」和「步长」

01: 相机

1.0 编辑器设置

将脚本编辑器中的「补全」，设置为「添加类型提示」：可以提高编辑器性能和编写的流畅

1.1 TileMap 快捷键

  1. ctrl + 左键点击 吸取单个图块
  2. ctrl + 按住左键拖动 吸取多个图块
  3. ctrl + shift + 按住左键拖动 绘制矩形区域
  4. 右键删除图块

1.2 相机位置

【步骤】在 player 节点下新建相机节点（这一步不是在 player 场景下设置的，而是在 world 场景下设置的）

【心得】安排节点的时候，要妥善确定位置：相机节点是实际游戏的 world 场景需要考虑的，player 自己不需要考虑，所以应该在world中设置（其实就是“怎么方便怎么来”，在一个场景修改再在另一个场景查看是不合适的）

【补充】拖动相机时，按住 ctrl 键可以更好的定位（位置辅助线提示）

1.3 相机跟随效果

游戏中相机并不总是跟随玩家，玩家在屏幕中心附近有一定的自由活动空间

  1. 在 Camera2D 节点的 Drag 属性勾选 Horizontal Enable和 Vertical Enable （水平和垂直方向上的相机拖动功能）
  2. 在 Camera2D 节点的 Editor 属性勾选 Draw Drag Margin，可以观察到可自由活动的范围，通过调整 Drag 属性的 Left Margin 等，可以控制其大小，值是 0 至 1 的比例

实现相机平滑移动，勾选 Camera2D 节点的 Position Smoothing 下的 Enabled

  1. 其中 Speed 可调整相机的平滑移动速度
